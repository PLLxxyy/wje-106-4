# 美食日记本 — 项目提示词

请根据以下要求生成项目代码和 README。

## 一、硬性约束

1. **纯前端项目**：无后端服务、无数据库，所有数据通过浏览器 IndexedDB 持久化存储。
2. **中文界面**：所有页面文案、按钮文字、提示信息均使用中文。
3. **技术栈不可替换**：React 18 + TypeScript + Zustand + Vite + Tailwind CSS + Chart.js（可选用 ECharts），状态管理仅用 Zustand，不得引入 Redux、MobX 等其他状态管理库。
4. **端口固定**：前端开发服务器端口映射为 `28606:5173`（宿主机 28606，容器内 5173）。
5. **COMPOSE_PROJECT_NAME**：`wjefooddiary`，Compose 顶层 `name` 字段必须显式声明此值。
6. **单文件行数上限**：任何源码文件不得超过 400 行，超过则必须拆分。
7. **禁止魔法数字/字符串**：所有枚举、常量、默认值必须抽取到 `constants/` 目录，禁止在业务代码中硬编码。
8. **IndexedDB 库**：使用 `dexie`（Dexie.js）封装 IndexedDB 操作，不得直接使用原生 IndexedDB API。
9. **图片存储**：食物照片以 base64 字符串形式存入 IndexedDB，上传前需压缩至宽度不超过 800px。
10. **组件单一职责**：每个组件文件只负责一个功能，严禁将多个不相关功能合并到同一文件。
11. **代码存放规则**：所有源码必须放在项目根目录下的 `frontend/src/` 中，严禁在根目录或 `frontend/` 根层级直接放置 `.ts` / `.tsx` 文件。
12. **Docker Compose 必须可用**：在任意目录名（包括中文目录名）下 `docker compose up -d` 均可正常启动。

---

## 二、项目概述

**美食日记本** 是一款面向个人用户的饮食记录与健康管理 Web 应用。用户可以按早/午/晚/加餐记录每餐所吃的食物，包含食物名称、份量、热量、营养素、照片和心情感受。应用提供日历视图回顾每日饮食、营养统计图表分析饮食趋势、食物库快速选择常用食物、以及饮食目标设定与达标提醒。所有数据存储在浏览器本地 IndexedDB 中，无需注册登录，即开即用。

**项目类型：纯前端 Web 应用（Static SPA）**

## 三、数据模型

### 3.1 DiaryEntry（饮食日记条目）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (uuid) | 主键 |
| date | string (YYYY-MM-DD) | 日期 |
| mealType | MealType | 餐次类型：breakfast / lunch / dinner / snack |
| foods | FoodRecord[] | 该餐包含的食物记录列表 |
| photo | string (base64, optional) | 该餐照片 |
| mood | MoodType | 心情：great / good / normal / bad |
| note | string | 文字感受/备注 |
| createdAt | string (ISO 8601) | 创建时间 |

### 3.2 FoodRecord（单条食物记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (uuid) | 主键 |
| name | string | 食物名称 |
| portion | number | 份数 |
| unit | string | 单位（如：份、碗、个、ml） |
| calories | number | 热量（千卡） |
| protein | number | 蛋白质（克） |
| fat | number | 脂肪（克） |
| carb | number | 碳水化合物（克） |
| photoAnnotation | AnnotationData (optional) | 照片标注数据（坐标+文字+价格） |

**AnnotationData 结构**：
```typescript
interface AnnotationData {
  annotations: Array<{
    x: number;        // 标注点 x 坐标（百分比 0-100）
    y: number;        // 标注点 y 坐标（百分比 0-100）
    label: string;    // 食物名称
    price?: number;   // 价格（可选）
  }>;
}
```

### 3.3 FoodItem（食物库条目）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (uuid) | 主键 |
| name | string | 食物名称 |
| category | FoodCategory | 分类：主食 / 肉类 / 蔬菜 / 水果 / 饮品 / 零食 / 其他 |
| caloriesPerPortion | number | 每份热量（千卡） |
| defaultUnit | string | 默认单位 |
| proteinPerPortion | number | 每份蛋白质（克） |
| fatPerPortion | number | 每份脂肪（克） |
| carbPerPortion | number | 每份碳水（克） |

### 3.4 DailyGoal（每日饮食目标）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (uuid) | 主键（固定为 'default'） |
| dailyCalories | number | 每日热量目标（千卡） |
| proteinTarget | number | 蛋白质目标（克） |
| fatTarget | number | 脂肪目标（克） |
| carbTarget | number | 碳水目标（克） |

### 3.5 Dexie 数据库 Schema

```typescript
// db/database.ts
const db = new Dexie('FoodDiaryDB');
db.version(1).stores({
  diaryEntries: '++id, date, mealType, createdAt',
  foodItems: '++id, name, category',
  dailyGoal: 'id',
});
```

## 四、核心页面（≥5个）

### 4.1 首页仪表盘（Dashboard）
- 今日热量摄入总览（CalorieRing 环形进度图）
- 今日三大营养素进度条（NutrientBar）
- 今日各餐次记录列表（FoodRecordCard）
- 快捷入口：记录饮食、查看日历、统计分析、食物库
- 饮食目标完成度提示（达标绿色/超标红色）

### 4.2 记录饮食页（DiaryCreate）
- 选择日期（默认今天）和餐次类型（MealType 枚举）
- 从食物库快速选择或手动输入食物
- 食物列表管理：添加/删除/修改份量
- 上传照片 + PhotoAnnotator 照片标注组件（标注食物名称和价格）
- 心情选择（MoodEmoji 组件）
- 文字备注输入
- 保存到 IndexedDB

### 4.3 饮食详情页（DiaryDetail）
- 展示某一天某一餐的完整记录
- 照片查看（含标注）
- 食物列表与营养素信息
- 编辑/删除操作

### 4.4 日历视图（Calendar）
- 月历形式展示当月每天的饮食摘要
- 每天单元格显示：餐次数、总热量、心情 emoji
- 点击某天展开 DayDetail 面板，展示当天所有餐次详情
- 左右切换月份

### 4.5 食物库管理（FoodLibrary）
- 食物列表：按分类筛选（FoodCategory 枚举）
- 搜索功能：按名称关键字搜索
- 新增/编辑/删除食物条目
- 预置 30 种常见食物种子数据

### 4.6 统计分析页（Statistics）
- 热量摄入趋势折线图（CalorieTrend）：按周/月切换
- 三大营养素占比饼图（NutrientPie）
- 每日餐次热量柱状图（MealBarChart）
- 时间范围选择器

### 4.7 目标设置页（Goals）
- 设置每日热量目标
- 设置三大营养素目标
- 保存到 IndexedDB
- 当前设置展示

### 4.8 设置页（Settings）
- 主题切换（亮/暗模式）
- 数据导出（导出全部数据为 JSON 文件）
- 数据导入（从 JSON 文件恢复数据）
- 清空所有数据（二次确认）

## 五、横切关注点

### 5.1 主题切换（亮/暗模式）

**触达层**：
- `useThemeStore`（Zustand）：管理主题状态（light/dark），持久化到 localStorage
- Tailwind CSS `dark:` 前缀：所有组件适配暗色模式
- CSS 变量（`styles/theme.css`）：定义 `--bg-primary`、`--text-primary`、`--border-color` 等
- Chart.js 主题适配（`utils/chartConfig.ts`）：根据主题切换图表颜色配置
- `<ThemeToggle>` 组件：放置在页面头部，一键切换

**实现要求**：
- 切换时在 `<html>` 标签上切换 `dark` class
- 所有颜色使用 Tailwind 的 `dark:` 变体或 CSS 变量
- 图表组件在主题切换时自动重新渲染

### 5.2 数据持久化与错误恢复

**触达层**：
- IndexedDB 封装层（`db/database.ts`）：Dexie 实例 + 表定义
- Zustand store（`useDiaryStore`、`useFoodStore`、`useGoalStore`）：读写 IndexedDB，内存缓存
- 导入导出 JSON（`utils/storage.ts`）：全量数据序列化/反序列化
- 全局错误提示组件（`<ErrorToast>`）：操作失败时弹出友好提示
- 初始化时自动加载种子数据（首次打开应用）

**实现要求**：
- 每个 Zustand store 自带 `loadFromDB()`、`saveToDB()` 方法
- 写入失败时显示错误提示，不丢失用户当前输入
- 导出 JSON 包含 `diaryEntries`、`foodItems`、`dailyGoal` 三个表
- 导入时覆盖写入，并刷新所有 store 状态

## 七、共享组件

### 7.1 `<CalorieRing>` — 热量环形进度图

- **功能**：以环形进度条形式展示当前热量摄入占目标的百分比
- **Props**：`current: number`（当前摄入）、`target: number`（目标热量）、`size?: 'sm' | 'md' | 'lg'`
- **样式**：使用 SVG 绘制圆环，达标（≤100%）显示绿色，超标（>100%）显示红色
- **使用页面**：首页仪表盘 / 日历详情 / 统计页面

### 7.2 `<FoodRecordCard>` — 食物记录卡片

- **功能**：展示单条食物记录的卡片组件
- **Props**：`record: FoodRecord`、`showPhoto?: boolean`、`onDelete?: () => void`
- **展示内容**：食物名称、份量、热量、三大营养素、照片缩略图（可选）
- **使用页面**：记录列表 / 日历详情

### 7.3 `<NutrientBar>` — 营养素进度条

- **功能**：展示单一营养素的摄入进度条
- **Props**：`label: string`（营养素名称）、`current: number`、`target: number`、`unit: string`、`color: string`
- **样式**：进度条颜色，超过 100% 变红警示
- **使用页面**：首页仪表盘 / 食物详情 / 目标设置页

### 7.4 `<MoodEmoji>` — 心情表情选择器

- **功能**：选择当前餐次的心情状态
- **Props**：`value: MoodType`、`onChange: (mood: MoodType) => void`、`readonly?: boolean`
- **样式**：四个 emoji 横向排列，选中状态放大并加边框
- **使用页面**：记录表单 / 日历视图

### 7.5 `<ThemeToggle>` — 主题切换按钮

- **功能**：一键切换亮/暗模式
- **使用页面**：全局页面头部

### 7.6 `<EmptyState>` — 空状态占位

- **功能**：列表无数据时显示空状态插图和提示文字
- **Props**：`message: string`、`actionLabel?: string`、`onAction?: () => void`
- **使用页面**：日历详情（无记录时）、食物库（无结果时）、统计（无数据时）

## 十、核心流程

### 流程 1：记录饮食
1. 用户进入「记录饮食」页面
2. 选择日期和餐次类型（早/午/晚/加餐）
3. 从食物库选择食物或手动输入（名称、份量、热量、营养素）
4. 可选：上传食物照片 → 打开 PhotoAnnotator 标注食物名称和价格
5. 选择今日心情（MoodEmoji）
6. 填写文字感受/备注
7. 点击保存 → 写入 IndexedDB → 返回首页

### 流程 2：查看日历
1. 进入「日历」页面，显示当月日历
2. 每天单元格显示餐次数、总热量、心情 emoji
3. 点击某天 → 展开 DayDetail 面板
4. DayDetail 展示当天所有餐次的 FoodRecordCard 列表
5. 点击某条记录 → 跳转 DiaryDetail 详情页

### 流程 3：统计分析
1. 进入「统计」页面
2. 选择时间范围（本周/本月/自定义）
3. CalorieTrend 折线图展示每日热量变化
4. NutrientPie 饼图展示三大营养素占比
5. MealBarChart 柱状图展示各餐次热量分布

### 流程 4：设置目标
1. 进入「目标设置」页面
2. 输入每日热量目标和三大营养素目标
3. 保存 → 写入 IndexedDB
4. 返回首页 → CalorieRing 和 NutrientBar 根据新目标重新计算进度

### 流程 5：达标检查
1. 首页仪表盘实时计算当日摄入 vs 目标
2. CalorieRing 环形图：绿色=达标（≤100%），红色=超标（>100%）
3. NutrientBar 进度条：每项营养素独立判断，超标部分变红
4. 超标时显示提示文字「今日热量已超标 XXX 千卡」

## 十二、代码存放规则

1. **页面组件**：`frontend/src/pages/` — 每个页面一个文件，文件名与路由对应
2. **通用组件**：`frontend/src/components/common/` — 跨页面复用的 UI 组件
3. **业务组件**：`frontend/src/components/diary/`、`chart/`、`calendar/` — 按功能域分目录
4. **状态管理**：`frontend/src/stores/` — 每个 store 独立文件，按业务域划分
5. **类型定义**：`frontend/src/types/` — 按数据模型拆分文件
6. **常量枚举**：`frontend/src/constants/` — 所有魔法值集中管理
7. **数据库层**：`frontend/src/db/` — Dexie schema 和种子数据
8. **Hooks**：`frontend/src/hooks/` — 自定义 hooks，每个 hook 独立文件
9. **工具函数**：`frontend/src/utils/` — 纯函数工具，按职责拆分
10. **样式文件**：`frontend/src/styles/` — 全局 CSS 变量和主题样式

**禁止事项**：
- 禁止在 `App.tsx` 中写业务逻辑，只负责路由组装和全局 Provider
- 禁止在一个文件中混合多个 Zustand store
- 禁止将页面组件和通用组件放在同一目录
- 禁止在组件文件中直接操作 IndexedDB，必须通过 store 层

## 十三、部署要求

### Docker Compose 一键部署

```bash
cp .env.example .env
docker compose up -d --build
```

访问地址：`http://localhost:28606`

### docker-compose.yml 参考骨架

```yaml
name: wjefooddiary

services:
  frontend:
    build: .
    container_name: ${COMPOSE_PROJECT_NAME:-wjefooddiary}-frontend
    restart: unless-stopped
    ports:
      - "${FRONTEND_PORT:-28606}:80"
```

### Dockerfile 参考骨架（多阶段构建）

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY frontend/package*.json ./
RUN npm ci
COPY frontend/ .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY frontend/nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

### frontend/nginx.conf 参考

```nginx
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
```

### .env.example

```dotenv
COMPOSE_PROJECT_NAME=wjefooddiary
FRONTEND_PORT=28606
```

### 本地开发

```bash
cd frontend
npm install
npm run dev    # 访问 http://localhost:5173
```

## 十七、Chart.js 配置要求

- 图表组件必须响应主题切换，使用 `useChartTheme` hook 获取当前主题的颜色配置
- 亮色模式：白色背景、深灰色文字、柔和彩色数据系列
- 暗色模式：深色背景、浅灰色文字、明亮彩色数据系列
- 图表需支持响应式布局（`responsive: true`）
- 折线图需有平滑曲线（`tension: 0.3`）
- 饼图需显示百分比标签
- 柱状图需有数值标注
